FPGA Platformer Game

Designed an FPGA-based game system with a custom SoC, HDMI video pipeline, and responsive low-latency controls.

Overview

This project was a custom FPGA platformer game built as a hardware/software co-design system. I designed the RTL for a custom SoC and integrated it with C-based software to support real-time gameplay, video output, and user input. The project combined digital design, embedded software, and system-level integration.

Demo

Problem / Goal

The goal was to build a complete interactive system on FPGA hardware rather than just isolated digital modules. I wanted to create something that required coordination across video generation, memory-mapped communication, input handling, and software control, while still meeting real-time performance goals.

Technical Implementation

  • Designed RTL for a custom FPGA-based SoC
  • Built an FSM-based HDMI video pipeline for display output
  • Implemented an SPI input controller for low-latency user interaction
  • Used AXI4-Lite for memory-mapped communication between components
  • Integrated hardware modules with C-based software control
  • Achieved real-time video processing at 30 FPS

Challenges

One of the hardest parts was coordinating timing across multiple hardware components while keeping the system responsive. Video pipelines are very sensitive to timing, and input handling needed to feel immediate. I had to carefully structure the hardware/software boundary so the game remained both stable and low-latency.

Results

The final system demonstrated real-time gameplay with 30 FPS output and input latency below 5 ms. It showed that the hardware and software stack could work together smoothly in an interactive application.

What I Learned

This project strengthened my skills in RTL design, SoC architecture, hardware/software integration, and designing digital systems under real-time constraints.
