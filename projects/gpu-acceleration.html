Parallel Programming for CNN Acceleration

Optimized CNN inference workloads on GPU using custom CUDA kernels, shared memory, and thread-level parallelism.

Overview

This project focused on accelerating core machine learning workloads using GPU parallel programming. I implemented and optimized high-performance kernels for operations such as matrix multiplication and convolution, then applied those techniques to LeNet-5 CNN inference. The goal was to better understand how low-level GPU design choices affect throughput, latency, and overall performance in real workloads.

Demo

Problem / Goal

CNN workloads are computationally expensive, especially for repeated convolution operations. The goal of this project was to improve performance by taking advantage of GPU parallelism and memory hierarchy, while also learning how to profile and tune kernels at a low level.

Technical Implementation

  • Optimized matrix multiplication and convolution workloads using CUDA
  • Used shared memory to reduce redundant global memory accesses
  • Improved performance through kernel scheduling and thread-level concurrency
  • Implemented custom CUDA kernels for LeNet-5 inference
  • Profiled execution with NVIDIA Nsight to identify bottlenecks
  • Tuned the kernels to reduce convolution latency and improve throughput

Challenges

One of the biggest challenges was balancing parallelism with memory efficiency. A kernel can launch many threads, but performance still suffers if memory access patterns are inefficient. Profiling was also essential, since the best optimization opportunities were not always obvious from code alone.

Results

The final implementation improved overall throughput by 40% on key HPC-style workloads and reduced convolution latency by 2.3x for LeNet-5 inference. These results showed the impact of shared memory optimization, better scheduling, and careful profiling-driven tuning. :contentReference[oaicite:1]{index=1}

What I Learned

This project gave me practical experience with CUDA programming, GPU memory hierarchy, performance profiling, and parallel optimization. It also helped me understand how systems-level thinking applies to machine learning workloads, especially when translating high-level models into efficient low-level implementations.
