Custom RISC-V Operating System

Built a Unix-like operating system for RISC-V with multitasking, virtual memory, syscalls, and low-level device drivers.

Overview

This project is a custom operating system built for the RISC-V architecture in C and RISC-V assembly. The goal was to understand operating systems at a much deeper level by implementing core kernel subsystems directly, including scheduling, memory management, interrupt handling, and hardware-facing drivers. Instead of only studying these ideas in class, I wanted to build them from the ground up and see how they interact in a real system.

Demo

Problem / Goal

Modern operating systems hide a huge amount of complexity behind abstractions. I wanted to build a smaller, Unix-like system that exposed the core ideas more directly: how processes are scheduled, how syscalls cross the user-kernel boundary, how virtual memory is mapped, and how devices are managed through memory-mapped I/O.

Technical Implementation

  • Implemented multitasking and scheduling for up to 16 processes
  • Built support for threading, context switching, and syscall handling
  • Implemented page table management and virtual memory mechanisms
  • Developed kernel subsystems including a file system and interrupt handling
  • Interfaced directly with hardware using memory-mapped I/O
  • Built device drivers for UART, RTC, and VirtIO

Challenges

One major challenge was managing interactions between multiple kernel subsystems at once. Context switching, interrupts, and memory management all affect each other, so debugging required careful reasoning about system state. Another challenge was working at the hardware boundary, where small mistakes in register usage or memory mapping could break the entire system.

Results

The project successfully supports multitasking, virtual memory, syscall execution, and direct device interaction. It gave me hands-on experience with the core mechanisms behind modern operating systems and significantly strengthened my low-level systems programming skills.

What I Learned

This project taught me how operating systems work beyond theory. I gained experience in kernel design, hardware/software interfacing, debugging low-level code, and structuring large systems where many components depend on each other.

Tech Stack
